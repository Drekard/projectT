## 🚀 Быстрый старт

### Что нужно для тестирования:

- **Два компьютера** в одной WiFi сети
- **Установленный Go** (версия 1.21+)
- **ProjectT** склонированный на оба устройства

### Шаг 1: Подготовка

На **обоих компьютерах** выполните:

```bash
cd c:\Users\egors\Desktop\projectT
go build -o p2p_test.exe ./cmd/p2p_test/
```

### Шаг 2: Запуск на ПК 1

```bash
./p2p_test.exe --node=1 --port=4001 --dht=true
```

Запомните адрес который появится в консоли, например:
```
projectt://QmABC123...@/ip4/192.168.1.5/tcp/4001/p2p/QmABC123...
```

### Шаг 3: Запуск на ПК 2

```bash
./p2p_test.exe --node=2 --port=4001 --dht=true --connect=<адрес_ПК_1>
```

Для диагностики проблем с подключением создана специальная утилита `p2p_diag` с пошаговым логированием.

#### Запуск на ПК 1 (сервер):

```bash
go run cmd/p2p_diag/main.go --mode=host --port=4001 --dht=true --advertise
```

#### Запуск на ПК 2 (клиент):

```bash
go run cmd/p2p_diag/main.go --mode=client --port=4001 --dht=true --target="<адрес_ПК_1>"
```

### Команды в p2p_diag:

- `/peers` - показать подключённые пиры с адресами
- `/connect <addr>` - подключиться с полной диагностикой
- `/advertise` - рекламировать себя в DHT
- `/find` - найти пиры через DHT с логом
- `/info` - показать информацию об узле
- `/log <text>` - записать сообщение в лог

## 🔍 Тестирование DHT

DHT (Distributed Hash Table) — распределённая хеш-таблица для обнаружения пиров в глобальной сети.

### Тест 1: Прямое подключение

**ПК 1:**
```bash
go run cmd/p2p_test/main.go --node=1 --port=4001 --dht=true
```

**ПК 2:**
```bash
go run cmd/p2p_test/main.go --node=2 --port=4001 --dht=true --connect=<адрес_ПК_1>
```

**Проверка:**
- На ПК 2 введите `/peers` — должен показать 1 подключённый пир
- На ПК 1 введите `/peers` — должен показать 1 подключённый пир

### Тест 2: Обнаружение через DHT

**ПК 1:**
```bash
go run cmd/p2p_test/main.go --node=1 --port=4001 --dht=true
# После запуска введите:
/advertise
```

**ПК 2:**
```bash
go run cmd/p2p_test/main.go --node=2 --port=4001 --dht=true
# После запуска введите:
/discovery
```

**Проверка:**
- ПК 2 должен найти ПК 1 через DHT
- В консоли появится сообщение об обнаруженном пире

### Тест 3: Чат между пирами

После подключения:

**ПК 1:**
```
Привет, ПК 2!
```

**ПК 2:**
```
Привет, ПК 1! Как дела?
```

**Проверка:**
- Сообщения должны приходить с префиксом `💬 [PeerID]`
- Оба пира получают сообщения друг друга

## 🌐 Как пиры находят друг друга

### Механизмы обнаружения пиров:

#### 1. Прямое подключение по адресу

**Как работает:**
- Вы вручную копируете адрес `projectt://PeerID@multiaddr` с одного ПК на другой
- Клиент парсит адрес и подключается напрямую

**Когда использовать:**
- ✅ Два компьютера в одной сети
- ✅ Тестирование на одном компьютере
- ✅ Когда известен точный адрес пира

**Пример адреса:**
```
projectt://QmABC123...@/ip4/192.168.1.5/tcp/4001/p2p/QmABC123...
```

**Преимущества:**
- Работает всегда если адрес верный
- Не требует bootstrap узлов
- Мгновенное подключение

**Недостатки:**
- Нужно вручную копировать адрес
- Адрес может измениться при перезапуске

---

#### 2. DHT Discovery (распределённая хеш-таблица)

**Как работает:**
1. ПК 1 рекламирует себя: `/advertise`
2. ПК 2 ищет пиры: `/discovery` или `/find`
3. DHT возвращает список пиров с адресами
4. ПК 2 подключается к найденным пирам

**Когда использовать:**
- ✅ Пиры в глобальной сети (интернет)
- ✅ Когда пиры не знают адреса друг друга
- ✅ Для автоматического обнаружения

**Важно:**
- DHT требует времени на распространение информации (30-60 сек)
- Без bootstrap узлов DHT работает только в локальной сети
- Для глобальной сети нужны bootstrap пиры

**Пример использования:**
```bash
# ПК 1 - рекламируем себя
go run cmd/p2p_diag/main.go --mode=host --advertise

# ПК 2 - ищем пиры
go run cmd/p2p_diag/main.go --mode=client --find
```

**Преимущества:**
- Автоматическое обнаружение
- Не нужно копировать адреса
- Работает в глобальной сети

**Недостатки:**
- Медленный старт (DHT bootstrap)
- Без bootstrap узлов может не найти пиры
- Требует больше ресурсов

---

#### 3. Bootstrap узлы

**Как работает:**
- При старте узел подключается к известным bootstrap пирам
- Через bootstrap пиры находит других пиров в DHT
- Bootstrap узлы хранятся в БД

**Публичные bootstrap узлы libp2p:**
```
/dnsaddr/relay.libp2p.io/p2p/QmNnooDu7bfjPFoTZYxMNLWUQJyrVwtbZg5gBMjTezGAJN
/dnsaddr/relay.libp2p.io/p2p/QmQCU2EcMqAqQPR2i9bChDtGNJchTbq5TbXJJ16u19uLTa
```

**Когда использовать:**
- ✅ Для подключения к глобальной сети
- ✅ Когда нужно найти много пиров
- ✅ Для стабильной работы DHT

**Как добавить bootstrap узел:**
```bash
# В приложении (когда будет реализовано)
/bootstrap add /dnsaddr/relay.libp2p.io/p2p/QmNnooDu7bfjPFoTZYxMNLWUQJyrVwtbZg5gBMjTezGAJN
```

---

### Сравнение механизмов:

| Механизм | Скорость | Надёжность | Когда использовать |
|----------|----------|------------|-------------------|
| Прямое подключение | Мгновенно | 100% | Локальная сеть, тестирование |
| DHT Discovery | 30-60 сек | 80% | Глобальная сеть |
| Bootstrap + DHT | 10-30 сек | 95% | Продакшн |

## 🛠 Диагностика проблем

### Проблема 1: Не подключается в локальной сети

**Симптомы:**
```
❌ Ошибка: context deadline exceeded
```

**Диагностика с p2p_diag:**

Запустите на ПК 2:
```bash
go run cmd/p2p_diag/main.go --mode=client --target="<адрес_ПК_1>"
```

**Лог успешного подключения:**
```
[15:30:46.796] 📍 Шаг 4: Попытка подключения...
[15:30:46.797]    ⏳ Таймаут: 30 секунд
[15:30:47.123]    ✅ ПОДКЛЮЧЕНИЕ УСПЕШНО (0.33 сек)
```

**Лог ошибки (таймаут):**
```
[15:30:46.796] 📍 Шаг 4: Попытка подключения...
[15:31:16.796]    ❌ ОШИБКА ПОДКЛЮЧЕНИЯ (30.00 сек): context deadline exceeded
   📋 Причина: истёк таймаут - узел недоступен или блокивается
```

**Лог ошибки (connection refused):**
```
[15:30:46.796] 📍 Шаг 4: Попытка подключения...
[15:30:46.850]    ❌ ОШИБКА ПОДКЛЮЧЕНИЯ (0.05 сек): connection refused
   📋 Причина: соединение отклонено - порт закрыт
```

**Решение:**

1. **Проверьте брандмауэр**
   ```bash
   # Windows - разрешить порт
   netsh advfirewall firewall add rule name="ProjectT P2P" dir=in action=allow protocol=TCP localport=4001
   ```

2. **Проверьте что оба ПК в одной сети**
   ```bash
   # ПК 1
   ipconfig | findstr IPv4

   # ПК 2
   ipconfig | findstr IPv4
   ```

   Адреса должны быть в одном диапазоне, например:
   - ПК 1: `192.168.1.5`
   - ПК 2: `192.168.1.10`

3. **Используйте полный multiaddr**
   ```
   /ip4/192.168.1.5/tcp/4001/p2p/QmABC123...
   ```

4. **Проверьте что сервер запущен**
   На ПК 1 должна быть строка:
   ```
   🏠 Режим СЕРВЕР - ожидание подключений
   ```

### Проблема 2: DHT не находит пиры

**Симптомы:**
```
/peers
Нет подключённых пиров
```

**Решение:**

1. **Подождите 30-60 секунд** — DHT медленно стартует

2. **Проверьте что DHT включён**
   ```bash
   --dht=true
   ```

3. **Используйте `/advertise` на ПК 1**
   ```
   /advertise
   ```

4. **Используйте `/discovery` на ПК 2**
   ```
   /discovery
   ```

5. **Проверьте логи** — должны быть сообщения:
   ```
   DHT обнаружение запущено
   Обнаружен новый пир: Qm...
   ```

### Проблема 3: NAT не пробивается

**Симптомы:**
- Подключение работает только в локальной сети
- Из внешней сети не подключается

**Решение:**

1. **Включите Hole Punching** (уже включён в коде)

2. **Используйте Relay** (уже включён)

3. **Настройте проброс портов** на роутере:
   - Port: 4001
   - Protocol: TCP
   - Internal IP: 192.168.1.5

4. **Добавьте bootstrap пиры** с публичными адресами

### Проблема 4: Паника в тестах (БД не инициализирована)

**Симптомы:**
```
panic: runtime error: invalid memory address or nil pointer dereference
```

**Решение:**

Это **ожидаемое поведение** для unit тестов которые не инициализируют БД.

Тесты с пометкой `t.Skip()` требуют инициализированную БД:
```
--- SKIP: TestP2PNetwork_ConnectionService/создание_сервиса_соединений
```

Для полноценного тестирования запустите приложение:
```bash
go run cmd/main.go
```

## 📊 Чек-лист проверки функциональности

### Базовая проверка

- [ ] Хост создаётся с уникальным PeerID
- [ ] PeerID сохраняется между перезапусками
- [ ] Можно экспортировать адрес в формате `projectt://PeerID@multiaddr`
- [ ] Можно подключиться к пиру по адресу

### DHT Discovery

- [ ] DHT инициализируется при старте
- [ ] `/advertise` рекламирует сервис в DHT
- [ ] `/discovery` находит другие пиры
- [ ] Пиры подключаются автоматически после обнаружения

### Connection Management

- [ ] Статус подключения отображается корректно
- [ ] KeepAlive отправляет ping каждые 30 секунд
- [ ] При разрыве соединения происходит переподключение
- [ ] После 3 неудачных ping пир помечается как offline

### Чат

- [ ] Сообщения отправляются через PubSub
- [ ] Сообщения принимаются от подключённых пиров
- [ ] Свои сообщения не дублируются в консоли
