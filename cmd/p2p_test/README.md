# 🧪 P2P Test Console

Консольная утилита для тестирования P2P соединений ProjectT.

## 📦 Сборка

```bash
# Скомпилировать
go build ./cmd/p2p_test/...

# Или запустить напрямую
go run cmd/p2p_test/main.go --node=1 --port=4001
```

## 🚀 Быстрый старт

### Тест 1: Два узла на одном компьютере

**Терминал 1 (Node 1):**
```bash
go run cmd/p2p_test/main.go --node=1 --port=4001
```

Запомните адрес, например:
```
/ip4/127.0.0.1/tcp/4001/p2p/QmABC123...
```

**Терминал 2 (Node 2):**
```bash
go run cmd/p2p_test/main.go --node=2 --port=4002 --connect=/ip4/127.0.0.1/tcp/4001/p2p/QmABC123...
```

### Тест 2: Два компьютера в одной сети

**ПК 1:**
```bash
go run cmd/p2p_test/main.go --node=1 --port=4001
```

Запомните полный адрес (будет показан при запуске).

**ПК 2:**
```bash
go run cmd/p2p_test/main.go --node=2 --port=4001 --connect=<адрес ПК 1>
```

### Тест 3: С DHT обнаружением

**ПК 1 (с рекламой):**
```bash
go run cmd/p2p_test/main.go --node=1 --port=4001 --dht=true
# После запуска введите: /advertise
```

**ПК 2 (с поиском):**
```bash
go run cmd/p2p_test/main.go --node=2 --port=4001 --dht=true
# После запуска введите: /discovery
```

## 📋 Флаги командной строки

| Флаг | Описание | Пример |
|------|----------|--------|
| `--node` | ID узла (1 или 2) | `--node=1` |
| `--port` | Порт для прослушивания | `--port=4001` |
| `--connect` | Адрес для подключения | `--connect=/ip4/.../p2p/Qm...` |
| `--dht` | Включить DHT обнаружение | `--dht=true` |
| `--mdns` | Включить mDNS обнаружение | `--mdns=false` |
| `--bootstrap` | Bootstrap узел | `--bootstrap=/dnsaddr/...` |

## 💬 Команды в консоли

| Команда | Описание |
|---------|----------|
| `/peers` | Показать подключённые пиры |
| `/connect <addr>` | Подключиться к пиру по адресу |
| `/discovery` | Найти пиры через DHT |
| `/advertise` | Рекламировать себя в DHT |
| `/info` | Показать информацию об узле |
| `/help` | Показать справку |
| `/quit` | Выход |
| **Любой текст** | Отправить сообщение в чат |

## 🎯 Примеры использования

### 1. Проверка NAT Traversal

```bash
# Узел 1
go run cmd/p2p_test/main.go --node=1 --port=4001 --dht=true

# Узел 2 (через интернет)
go run cmd/p2p_test/main.go --node=2 --port=4001 --dht=true --bootstrap=<адрес узла 1>
```

Если NAT пробился — пиры подключатся автоматически.

### 2. Тест чата

```bash
# Узел 1
go run cmd/p2p_test/main.go --node=1 --port=4001

# Узел 2
go run cmd/p2p_test/main.go --node=2 --port=4002 --connect=<адрес узла 1>

# В обоих терминалах вводите сообщения
Привет!
Как дела?
Тест чата работает!
```

### 3. Поиск пиров через DHT

```bash
# Узел 1 - рекламируем себя
go run cmd/p2p_test/main.go --node=1 --port=4001 --dht=true
# В консоли: /advertise

# Узел 2 - ищем пиры
go run cmd/p2p_test/main.go --node=2 --port=4001 --dht=true
# В консоли: /discovery
```

## 🔍 Диагностика проблем

### Проблема: Не подключается в локальной сети

**Решение:**
1. Проверьте, что брандмауэр не блокирует порт
2. Убедитесь, что оба узла в одной сети
3. Попробуйте указать полный multiaddr

### Проблема: DHT не находит пиры

**Решение:**
1. Подождите 30-60 секунд (DHT медленно стартует)
2. Используйте `/advertise` на одном узле и `/discovery` на другом
3. Проверьте, что `--dht=true` на обоих узлах

### Проблема: NAT не пробивается

**Решение:**
1. Включите `EnableHolePunching()` в настройках
2. Используйте relay-узлы
3. Настройте проброс портов на роутере

## 📊 Что проверяется

| Функция | Как проверить |
|---------|--------------|
| **Подключение** | `/peers` — должен быть 1 пир |
| **KeepAlive** | Соединение не рвётся в простое |
| **Чат** | Отправка сообщений между узлами |
| **DHT** | `/discovery` находит пиры |
| **NAT Traversal** | Подключение через интернет |
| **AutoRelay** | Работа за симметричным NAT |

1. **Не закрывайте окно** — нажмите `/quit` для корректного завершения
